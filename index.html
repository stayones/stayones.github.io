8.5 Leetcode

Path Sum II

Given the root of a binary tree and an integer targetSum, return all root-to-leaf paths where each path’s sum equals targetSum.

A leaf is a node with no children.

Constraints:

  • The number of nodes in the tree is in the range [0, 5000].
  • -1000 <= Node.val <= 1000
  • -1000 <= targetSum <= 1000

解题思路：

深度优先遍历整棵树，将树中从根节点到叶节点的所有路径都保存在临时结果数组中，如果发现整条路径的和与targetSum相匹配，则将该条路径放入最终的结果数组中。

Recommend Sys Learning points

候选生成的三种判断方式（相似性计算）：

  1. cosine：角度越接近的越类似；
  2. dot：向量点乘s(q,x)=∣∣q∣∣∣∣x∣∣cos(q,x)s(q,x)=||q||||x||cos(q,x)s(q,x)=∣∣q∣∣∣∣x∣∣cos(q,x)
  3. Euclidean distance：欧氏几何距离

使用dot来计算相似性，可能会使得受欢迎的物品被更多的推荐，因为该item的norm会更大，所以有时候也会采用consine的方式来计算相似性

基于内容的过滤

  1. 将不同商品用向量表示，将该用户的特征用向量表示
  2. 采用dot方式计算商品与用户间的相似性，选择dot后乘积最大的作为推荐的物品

协同过滤

  1. 矩阵分解的方式：

    1. user embedding和item embedding相乘得到用户对每个物品预测的喜爱度

    2. 与原本观察得到的feedback矩阵相比较（0-1矩阵–用户看过标记为1，没有看过标记为0）

    3. 可以采用SVD算法，最小化误差函数，来更新user和item的ebedding

    4. 可以使用SGD（Stochastic gradient descent）来最小化损失函数；

    5. 或者使用WALS（weighted alternating least quares）
